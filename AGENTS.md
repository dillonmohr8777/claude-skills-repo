# AGENTS.md — Codex instructions for the Claude skills repo

This repository is a library of **Claude Skills** (each `skills/<name>/SKILL.md`).
Codex CLI sessions running in this repo can use any skill below as a playbook.

## How to use a skill (Codex)

1. Match the user's request to a skill in the catalog below by its description.
2. Read that skill's full file: `skills/<name>/SKILL.md`.
3. Follow the procedure described in that file. If the skill links to
   `references/*.md` files inside its own directory, read those as needed.
4. The YAML frontmatter at the top of each `SKILL.md` defines the trigger
   conditions (`description`) and may pin specific tools (`allowed-tools`).
   Treat that frontmatter as authoritative.

## Conventions

- Each skill directory is self-contained. Anything it needs (templates,
  references, scripts, evaluators) lives under `skills/<name>/`.
- Some skills have their own `AGENTS.md` or `.codex/instructions.md` — those
  override this top-level file when Codex is run from inside that directory.
- Non-skill support directories (`agents/`, `commands/`, `docs/`,
  `documentation/`, `scripts/`, `standards/`, `templates/`, `tests/`,
  `orchestration/`, `eval-workspace/`, `custom-gpt/`) are tooling, not skills.

## Adjacent resources Codex can pull from

Beyond the SKILL.md files indexed below, this repo also contains:

- `skills/agents/` — agent-persona prompts organized by function
  (`business-growth/`, `c-level/`, `engineering/`, `engineering-team/`,
  `finance/`, `marketing/`, `personas/`, `product/`, `project-management/`,
  `ra-qm-team/`). Use one of these as a system prompt when the user asks
  for a domain expert (e.g. "act as a CTO advisor" → read
  `skills/agents/c-level/cs-cto-advisor.md`).
- `skills/commands/` — slash-command-style playbooks (`a11y-audit.md`,
  `code-to-prd.md`, `prd.md`, `retro.md`, `rice.md`, `seo-auditor.md`,
  `sprint-plan.md`, `tc.md`, etc.). These are reusable prompts; read the
  matching file when the user invokes the command by name.
- `skills/standards/` — repo-wide quality, communication, git, and
  documentation standards. Treat as authoritative house style.

## Cross-model handoffs

The `codex` skill itself documents how Claude Code delegates work to Codex
(via `codex-plugin-cc` or the `codex` CLI). The reverse direction — Codex
delegating back to Claude — isn't wired up here; if you need a Claude opinion
from a Codex session, ask the human to run it in Claude Code.
